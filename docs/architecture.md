# Architecture (Nuxt edition)

In-depth notes on how the Nuxt edition is wired together. Pair with `CLAUDE.md` / `context.md` for the high-level orientation, and with `git log` for *why*. The engine half is unchanged from the upstream [`manuelhintermayr/sketchbook-upgraded`](https://github.com/manuelhintermayr/sketchbook-upgraded) Webpack edition — see the heritage note in [`../README.md`](../README.md). What this doc focuses on is the **engine ↔ Vue bridge**, the per-frame loop, and where the two halves meet.

## The engine ↔ Vue bridge

Engine code never imports `vue` or `~/composables`. Vue code never reaches into engine privates. They communicate through one file:

### `engine/state/index.ts`

```ts
export interface EngineStateBridge {
    loading: {
        setVisible(v: boolean): void;
        setProgress(p: number): void;
        setMessage(m: string): void;
    };
    hud: {
        setUiContainer(v: HTMLElement | null): void;
        setControlsOverlay(v: boolean): void;
        setFps(v: boolean): void;
        setDebugStack(v: HTMLElement | null): void;
        toggleControlsOverlay(): void;
    };
    race: { setLap(lap: number): void };
    scenario: {
        setOnMoon(v: boolean): void;
        setPlanetMenuOpen(v: boolean): void;
        setActiveScenarioId(id: string | null): void;
        setPlanetSelect(handler: ((target: 'earth' | 'moon') => void) | null): void;
    };
    scenarios: {
        register(entry: ScenarioEntry): void;
        clear(): void;
    };
    controls: { setRows(rows: ControlRow[]): void };
    startupModals: {
        showWelcome(): Promise<void>;
        showEmpty(): Promise<void>;
        showWebglWarning(): Promise<void>;
        showScenarioWelcome(title: string, body: string): Promise<void>;
    };
    pause: {
        setEnabled(v: boolean): void;
        setRestartHandler(h: (() => void) | null): void;
        setSettingsHandler(h: (() => void) | null): void;
    };
    dialog: { ... };
    proximity: { ... };
    iris: { open(): Promise<void>; close(): Promise<void> };
    error: { show(error: { code, title, stack }): void };
}

let impl: EngineStateBridge = STUB;
export function engineState(): EngineStateBridge { return impl; }
export function bindEngineState(next: EngineStateBridge): void { impl = next; }
```

The `STUB` is a no-op object with the same shape, so calling `engineState().pause.setEnabled(true)` before mount is safe (just does nothing).

### EngineHost.vue (the wire-up)

```ts
onMounted(() => {
    bindEngineState({
        loading: {
            setVisible: (v) => { loading.visible.value = v },
            setProgress: (p) => { loading.progress.value = p },
            setMessage: (m) => { loading.message.value = m },
        },
        scenario: {
            setOnMoon: (v) => { scenario.onMoon.value = v },
            setPlanetMenuOpen: (v) => { scenario.planetMenuOpen.value = v },
            setActiveScenarioId: (id) => { scenario.activeScenarioId.value = id },
            setPlanetSelect: (h) => { scenario.setPlanetSelect(h) },
        },
        scenarios: {
            register: (e) => scenarios.register(e),
            clear: () => scenarios.clear(),
        },
        // ... etc
    });
});

onUnmounted(() => bindEngineState(STUB));
```

### Why this shape

- **No circular imports.** Engine ↔ composables would create a graph cycle every TS / bundler hates.
- **Testable engine.** `engine/` runs without Vue. The stub means engine unit tests don't need a Vue instance.
- **Hot-reload safe.** When EngineHost remounts (HMR, route change), the old `impl` object is GC'd; new closures bind. The engine re-reads `engineState()` per call so it picks up the new closures the next frame.
- **Single grep target.** Every cross-component state operation goes through `engineState().<group>.<setter>` — `git grep engineState\(\)\.scenario` enumerates every scenario-related engine call.

## Update order (lower runs first)

Slot constants are defined in `engine/enums/UpdateOrder.ts`. Each slot is spaced by 10 so a new entry can squeeze between two existing ones without renumbering everything.

| Slot (value) | Class(es) |
|---|---|
| `CharacterPhysics` (10) | `Character` — physics step, state-machine update, control input |
| `VehiclePhysics` (20) | `Vehicle` — physics step, control input, hard-landing detection, stuck-recovery |
| `Input` (30) | `InputManager` — drains mouse/keyboard buffers, dispatches to receiver |
| `Camera` (40) | `CameraOperator` — orbit/free-cam input, position lerp |
| `Environment` (50) | `Sky` — sun position, day/night cycle, CSM frustum sync. `ShapeEntity` shares the slot |
| `Scenarios` (60) | `RaceContent` — per-frame plane crossings against checkpoints |
| `World` (100) | `Grass` (shader uniforms), `Ocean` (wave / normal-map), `WanderingAnimals`, `Birds`, `Butterflies` |
| `Audio` (110) | `ProceduralAudio` (engine + ambient + background music), `Speaker` |
| `Prompts` (130) | `ProximityPrompt` — no-op per frame (relies on TriggerCube + keydown) |
| `Labels` (140) | `WorldLabels` — distance-cull CSS2D name tags |
| `PostCamera` (150) | `CameraShake` — adds transient camera-position offset after CameraOperator finalises the frame's camera |

`updateOrder` is the IUpdatable contract; reactive `watch()`s and Scenario.launch don't run inside this loop.

## World construction sequence

```
new Sketchbook.World(scenePath)
  ├─ setupRendererPipeline(this)  ← renderer + composer + FXAA + resize
  ├─ bootstrapHTML(this)          ← document.body.appendChild(canvas)
  ├─ initStats()
  ├─ paramsBootstrap(this)        ← params object + persistence (no panel)
  ├─ scenario watchers + Debug_FPS watcher (immediate:true) installed
  ├─ new InputManager / CameraOperator / Sky / CameraShake / OutlineEffect /
  │    AmbientSound / WorldLabels
  └─ if scenePath:
        loadingManager = new LoadingManager(this)
        loadingManager.onFinishedCallback = () => {
            this.update(1, 1); this.setTimeScale(1);
            UIManager.setUserInterfaceVisible(true);
            engineState().pause.setEnabled(true);  // Esc now works
            engineState().startupModals.showWelcome();  // UX layer
        }
        if string: loadingManager.loadGLTF(path, gltf => loadScene(this, lm, gltf))
        else (BaseScene): loadScene(this, lm, {scene: instance.scene})
```

`loadScene(world, lm, gltf)` (in `engine/world/loading/SceneLoader.ts`) walks every node, branches on `userData`, and registers entities. `addMapSwitcher(world)` runs at the start so the map dropdown lands above the scenario buttons. After parsing the GLB, it calls `injectDefaultSceneNPCs(world)`, `injectWanderingAnimals(world)`, `injectFlyingBirds(world)`, `injectButterflies(world)`, then launches the default scenario.

## Lifecycle interfaces

```
IUpdatable           updateOrder: number
                     update(timeStep, unscaledTimeStep): void

IWorldEntity         extends IUpdatable
                     entityType: EntityType
                     addToWorld(world): void
                     removeFromWorld(world): void

ISpawnPoint          spawn(loadingManager, world): void

ICollider            options: any
                     body: CANNON.Body
```

`world.add(entity)` calls `addToWorld` and `registerUpdatable`. `world.remove(entity)` does the inverse.

## State machines

- **Character** — `engine/characters/character_states/`. ~25 states. State changes via `character.setState(new Walk(this))`. Each state has `onInputChange()` and `update()`.
- **Vehicle entry** — `VehicleEntryInstance` orchestrates the multi-frame walk-up + door-open + sit sequence.
- **AI behaviours** — `RandomBehaviour`, `FollowTarget`, `FollowPath`. Set via `character.setBehaviour(new FollowPath(node, speed))`.

## Physics

- One `CANNON.World` per `World`. Gravity scaled by `params.Gravity_Scale` (lunar mode = 1.62 m/s²).
- `physicsFrameRate = 60`, fixed `physicsFrameTime = 1/60`. Substeps via `world.step(...)` inside `World.update()`.
- Collision groups in `engine/enums/CollisionGroups.ts`.
- `CannonDebugRenderer` is wired (toggle via `params.Debug_Physics` from the DebugPanel).

## Rendering pipeline

Setup lives in `engine/world/setup/RendererPipeline.ts` and is called once from `World`'s constructor. Per-frame work runs from `World.render`.

- `THREE.WebGLRenderer` with PCF shadows, ACES tone mapping, `pixelRatio` capped at 2.
- `EffectComposer` chain: `RenderPass` → `FXAAShader`. FXAA can be toggled at runtime (`world.params.FXAA`); when off, `tickRenderPipeline` calls `renderer.render()` directly to bypass the composer pass cost.
- `OutlineEffect` runs *after* the composer. Two-pass: depth pre-pass into a `HalfFloatType` render target via `MeshDepthMaterial` override (skips `RenderLayer.OutlineSkip`), then a Sobel-edge fullscreen quad with a scale-invariant ratio threshold.
- `CSM` (cascaded shadow maps) attached to `Sky` with `shadowMapSize: 1024` × 3 cascades.
- `CSS2DRenderer` runs after the outline pass to project name-tag divs above their world-space anchor. Lives at `world.labelRenderer`, has its own absolutely-positioned overlay div with `pointer-events: none`. Distance culling is centralised through `WorldLabels`.
- GPU shader pre-compile: `LoadingManager.doneLoading` awaits `renderer.compileAsync(scene, camera)` before lifting the loading screen.

## CSS layout — the body siblings problem

The engine appends three direct `<body>` children: `<canvas id="canvas">`, `<div id="labelRenderer">`, and one `<div class="proximity-prompt">` per active prompt. They live outside `#__nuxt` so they aren't subject to its `pointer-events: none`.

`base.css` defines an explicit z-stack:

```
0  -  #canvas
1  -  #labelRenderer + .proximity-prompt
2  -  #__nuxt   (Vue UI overlay, position: fixed, pointer-events: none)
```

Without explicit z-indices the canvas would render on top of `__nuxt` (DOM order: canvas is appended after `__nuxt`) and hide the entire UI. With them, mouse events still reach the canvas because `__nuxt` is `pointer-events: none`.

**Don't pin `body` or `html` to `position: fixed/absolute`.** That moves the CSS2DRenderer's containing block onto `body`, which mis-anchors every NPC name label. The `#__nuxt` overlay pattern + explicit z-indices is what fixes the "white band along the top" symptom Nuxt UI's `<UApp>` flow blocks were causing — pin `__nuxt`, not the document.

When `html.touch-active` is set (driven by `useTouchMode`), `responsive.css` applies `touch-action: none; overscroll-behavior: none` to `html / body / #__nuxt / #canvas` so vertical drags on the joystick don't fire `pointercancel`.

## Audio

All audio modules live in `engine/world/audio/` and share a single `THREE.AudioContext.getContext()` so the browser's ~6-context limit is never an issue, regardless of vehicle count.

- `ProceduralAudio` is the abstract base for *continuous* synths (engine, ambient, background music). Subclasses provide `shouldPlay()`, `buildSynth()`, `teardownSynth()`, `updateSynth()`. The base handles the master-gain ramp, lazy AudioContext acquisition, and the lifecycle.
- `EngineSound` (per-Vehicle) has 5 timbre profiles (car / heli / airplane / boat / rocket) selected via `vehicle.engineSoundProfile`. RPM is modulated by chassis speed.
- `AmbientSound` — wind + water synth. Water gain gated by `Math.abs(cam.y - 12) < 10`.
- `BackgroundMusic` (extends `ProceduralAudio`) loops bundled music tracks; gated by `params.Background_Music` and scaled by `Master_Volume * Music_Volume`.
- `SfxBus` is event-driven (race checkpoint, dialog whoosh, prompt + pause UI ticks, iris-transition whoosh, vehicle crash, rocket boom).
- `CharacterSfx` (per-Character) — positional audio for footsteps / jump / land / door clunk attached to each character's body.
- `Speaker` — map-driven 3D positional audio source built from a `userData.data='speaker'` marker. Uses `createMediaAudioElement(url)` from `AudioHelpers`.
- `BirdSound` — per-bird `PositionalAudio` with FM-chirp synth on a 5–12 s Poisson chirp schedule.
- `AnimalVoices` — bark / meow / purr-loop bus for wandering animals; uses `StereoPannerNode` + manual distance-gain.
- `AudioHelpers` exports `getMasterVolume(world)` (returns 0 when `Master_Audio` is off), `ensureAudioListener(world)` (lazy-create THREE.AudioListener attached to camera), `createMediaAudioElement(url)`.
- `Master_Audio` mute switch routes through `World.applyAudioListenerVolume()` which sets the THREE listener gain to 0. Title-screen mute button + Settings modal toggle + DebugPanel all mirror through `localStorage['sketchbook.soundMuted']`.
- Browser autoplay-policy gating: every Speaker that fails to autoplay registers itself on a static queue; a single `pointerdown`/`keydown` listener on `window` plays everything queued.

## Persistence

- `localStorage['sketchbook-settings']` — full lil-gui state via `gui.save()` / `gui.load()`. Persisted on every change via `gui.onFinishChange`.
- `localStorage['sketchbook.map']` — selected map id. Read by `EngineHost.vue` before constructing World; written by the Map dropdown.
- `localStorage['sketchbook.locale']` — selected language (en / de / es). `useUserPrefs` reads + writes; `@nuxtjs/i18n`'s `setLocale` mirrors.
- `localStorage['sketchbook.soundMuted']` — `Master_Audio` flag. Title-screen mute button writes; ParamsBootstrap reads on next boot.
- `localStorage` keys mirror the upstream Webpack edition's names so users who try both keep their preferences.

## Sandbox scenes (BaseScene subclasses)

`engine/world/sandboxes/BaseScene.ts` is an abstract class with a `THREE.Scene` and three vehicle-mesh slots (kept for upstream compat). Subclasses (`TestScene`, `Test2Scene`, `Test3Scene`, `ExampleScene`) populate `this.scene` with meshes carrying the same userData markers as a `.glb`. `World` accepts either a string `.glb` path or a `BaseScene` instance.

## Map switcher

`addMapSwitcher(world)` adds map options to the scenario registry. Selecting one writes to `localStorage['sketchbook.map']` and reloads the page (covered by an iris-wipe transition). `EngineHost.vue` reads the value on next boot and dispatches to either `glbPaths[…]` or `new sceneClasses[…]()`.
